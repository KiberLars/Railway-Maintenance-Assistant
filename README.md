# RMA: Railway Maintenance Assistant

`RMA` (`Railway Maintenance Assistant`) - система интеллектуальной поддержки технического обслуживания объектов ВСМ (высокоскоростных магистралей).

MVP описывает Android-приложение для планшетов и смартфонов, которое по заявке или осмотру помогает определить объект (локальный OCR по шильдику), найти инструкцию в локальной полной базе знаний, получить обоснованную ИИ-подсказку (RAG и LLM) при наличии сети, при необходимости подключить эксперта по видеосвязи, провести чек-лист, зафиксировать результат и сформировать отчёт. Серверная часть управляет базой знаний, заявками, синхронизацией журналов, тяжёлыми AI-функциями (self-hosted LLM/RAG, STT/TTS) и контуром обучения.

## Основные решения MVP

- Целевая платформа - Android-планшеты (приоритет) и смартфоны.
- Предметная область - объекты ВСМ: путевая инфраструктура, устройства автоматики и связи, электротехнические объекты.
- AR/VR и EAM-интеграция исключены из проекта.
- OCR шильдика выполняется локально на устройстве.
- Полная база знаний кэшируется локально и доступна без сети.
- Чек-листы и журналы создаются локально; синхронизация выполняется через outbox с идемпотентностью.
- LLM/RAG, STT/TTS и видеосвязь работают только онлайн; LLM размещается self-hosted на GPU.
- Сложные случаи эскалируются эксперту по видеосвязи (человек в контуре).
- Разобранные случаи пополняют базу знаний после проверки (контур обучения).
- Backend разделён на сервисы под четырьмя логическими контурами: API Gateway/Auth, Documentation Service, Knowledge Sync Service, Search/RAG Service, Speech Service, Dispatch/Ticketing Service, Operation Log/Sync Service, Expert/Collaboration Service, Learning/Feedback Service, Admin Panel.

## Ключевые сущности и идентификаторы

- `device_id` - стабильный идентификатор объекта.
- `request_id` - идентификатор заявки о неисправности.
- `instruction_version` - версия инструкции или чек-листа.
- `knowledge_base_version` - версия локально кэшируемой базы знаний.
- `client_operation_id` - локальный идентификатор операции обслуживания.
- `operation_event_id` - идентификатор события журнала действий.
- `consultation_id` - идентификатор видеоконсультации с экспертом.
- `candidate_id` - идентификатор кандидата в базу знаний.
- `idempotency_key` - ключ для безопасной повторной синхронизации.
- `correlation_id` - идентификатор для связи логов, метрик и состояния процесса.

## Проверка перед сдачей

- Все Mermaid-диаграммы должны рендериться.
- Названия компонентов и сущностей должны совпадать во всех разделах.
- AR/VR и EAM не должны упоминаться как часть системы.
- Все существенные архитектурные решения должны быть отражены в ADR.

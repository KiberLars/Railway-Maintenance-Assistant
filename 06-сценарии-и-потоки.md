# 06. Сценарии и потоки

## Основной онлайн-сценарий

Сеть доступна: специалист получает задание по заявке, использует локальный OCR и базу знаний, а также онлайн-помощь RAG и Speech.

```mermaid
sequenceDiagram
    autonumber
    actor Disp as Диспетчер
    actor Tech as Специалист
    participant App as Мобильное приложение
    participant GW as API Gateway/Auth
    participant Sp as Speech
    participant RAG as Search/RAG
    participant Log as Operation Log/Sync

    Disp->>GW: Создать заявку о неисправности
    GW->>App: Задание назначено специалисту
    Tech->>App: Открывает задание, сканирует шильдик
    App->>App: Local OCR извлекает маркировку
    App->>App: Локальный поиск Device и Instruction
    Tech->>App: Описывает проблему (текст или голос)
    opt Голосовой ввод
        App->>+GW: Запрос STT
        GW->>Sp: Распознать речь
        Sp-->>GW: Текст проблемы
        GW-->>-App: Текст проблемы
    end
    App->>+GW: Онлайн-подсказка по проблеме
    GW->>RAG: Запрос RAG по базе знаний
    RAG-->>GW: Подсказка и ссылки на источники
    GW-->>-App: Ответ RAG (решение остаётся за человеком)
    Tech->>App: Выполняет чек-лист, фиксирует фото
    App->>App: Сохраняет MaintenanceJob и OperationLog
    App->>+GW: Отправляет outbox-события
    GW->>Log: Применить идемпотентно
    Log-->>GW: Ack по operation_event_id
    GW-->>-App: Синхронизация подтверждена
```

## Эскалация эксперту по видеосвязи

Если случай сложный, специалист подключает эксперта; ИИ только подсказывал, решение принимает человек.

```mermaid
sequenceDiagram
    autonumber
    actor Tech as Специалист
    participant App as Мобильное приложение
    participant GW as API Gateway/Auth
    participant Exp as Expert/Collaboration
    participant Media as Media-сервер
    actor ExpP as Эксперт

    Tech->>App: Сложный случай, запросить эксперта
    App->>+GW: Запрос видеоконсультации
    GW->>Exp: Создать сессию консультации
    Exp->>Media: Выделить медиаканал (SFU/TURN)
    Exp-->>GW: Сессия готова
    GW-->>-App: Параметры подключения
    par Видеопоток
        App->>Media: Видео специалиста
        ExpP->>Media: Видео эксперта
    end
    ExpP->>Exp: Правит рекомендацию, подтверждает решение
    Exp-->>App: Обновлённая рекомендация
    App->>App: Фиксирует решение в MaintenanceJob
    Note over Tech,ExpP: Финальное решение остаётся за человеком
```

## Офлайн-сценарий с последующей синхронизацией

Базовый сценарий не зависит от сети: вся база знаний уже на устройстве, тяжёлые онлайн-функции недоступны.

```mermaid
sequenceDiagram
    autonumber
    actor Tech as Специалист
    participant App as Мобильное приложение
    participant GW as API Gateway/Auth
    participant Log as Operation Log/Sync

    Tech->>App: Сканирует объект без сети
    App->>App: Local OCR извлекает маркировку
    App->>App: Локальный поиск Device, Instruction, Checklist
    App-->>Tech: ИИ-подсказка, Speech и видео недоступны
    Tech->>App: Выполняет чек-лист, фиксирует фото
    App->>App: Создаёт MaintenanceJob, события в outbox (pending)
    loop Пока нет сети
        App->>App: Хранит события со статусом pending
    end
    App->>+GW: При появлении сети отправляет outbox
    GW->>Log: Применить по idempotency_key
    Log-->>GW: Ack новых и уже применённых событий
    GW-->>-App: Результат синхронизации
    App->>App: Обновляет статусы событий
```

## Контур обучения

Разобранный случай (особенно после правок эксперта) становится кандидатом в базу знаний и публикуется только после проверки человеком.

```mermaid
sequenceDiagram
    autonumber
    participant App as Мобильное приложение
    participant GW as API Gateway/Auth
    participant Learn as Learning/Feedback
    actor Rev as Эксперт или администратор
    participant Doc as Documentation
    participant Sync as Knowledge Sync

    App->>+GW: Завершённый кейс
    GW->>+Learn: Сформировать кандидата в базу знаний
    Learn-->>-GW: Кандидат на модерации
    GW-->>-App: Принято
    Rev->>Learn: Проверяет кандидата
    alt Одобрен
        Learn->>Doc: Публикация в контент
        Doc->>Sync: Новая knowledge_base_version
        Sync-->>App: Обновление доступно
    else Отклонён
        Learn-->>Rev: Кандидат отклонён с комментарием
    end
```

## Обновление базы знаний

1. Администратор меняет объекты, инструкции или чек-листы в Admin Panel; одобренные кандидаты из контура обучения тоже попадают в контент.
2. Documentation Service сохраняет черновик и после публикации создаёт новую `knowledge_base_version`.
3. Knowledge Sync Service готовит пакет полной базы знаний или инкрементальное обновление.
4. Мобильное приложение периодически проверяет доступную версию.
5. Клиент скачивает обновление, проверяет контрольную сумму и применяет миграцию локального хранилища.
6. Уже начатые операции продолжают ссылаться на старую `instruction_version`.
7. Новые операции используют актуальную версию базы знаний.

## Жизненный цикл MaintenanceJob

```mermaid
%%{init: {'flowchart': {'curve': 'stepBefore'}}}%%
flowchart TD
    Start([Старт]) -->|операция начата| Created[created]
    Created -->|объект определён OCR или вручную| Identified[identified]
    Identified -->|инструкция найдена| Loaded[instruction_loaded]
    Loaded -->|чек-лист начат| Progress[in_progress]
    Progress -->|запрос эксперта| Escalated[escalated]
    Escalated -->|рекомендация получена| Progress
    Progress -->|приостановлено| Paused[paused]
    Paused -->|продолжено| Progress
    Progress -->|чек-лист завершён| Completed[completed]
    Progress -->|невозможно выполнить| Failed[failed]
    Completed -->|события в outbox| SyncPending[sync_pending]
    Failed -->|события в outbox| SyncPending
    SyncPending -->|сервер подтвердил| Synced[synced]
    SyncPending -->|ошибка синхронизации| SyncFailed[sync_failed]
    SyncFailed -->|повторная отправка| SyncPending
    Synced --> End([Завершено])
```

## Правила повторов

| Повтор | Правило |
|---|---|
| Повторная отправка `OperationLog` | Сервер применяет событие один раз по `operation_event_id` и `idempotency_key` |
| Повторная загрузка обновления базы знаний | Клиент проверяет `knowledge_base_version` и контрольную сумму |
| Повторный OCR одного шильдика | Создаёт новую попытку распознавания внутри той же операции |
| Повторный онлайн-RAG запрос | Не меняет состояние операции, сохраняется как подсказка |
| Повторное подключение к эксперту | Создаёт новую сессию консультации, не меняя историю операции |
| Повторная отправка кандидата в базу знаний | Learning/Feedback дедуплицирует кандидата по исходному кейсу |

## Ошибочные и альтернативные ветки

- Если OCR не уверен в результате, специалист подтверждает объект из заявки или вручную.
- Если локальная база знаний повреждена, приложение блокирует начало новых операций и требует повторной загрузки базы знаний.
- Если RAG или Speech недоступны, остаётся локальный поиск и ручной текстовый ввод.
- Если эксперт недоступен, специалист продолжает по локальной инструкции или откладывает сложный случай; запрос консультации остаётся в ожидании.
- Если синхронизация вложения не удалась, `OperationLog` остаётся в outbox до успешной отправки.
- Если сервер вернул конфликт версии, клиент не меняет завершённую операцию, а сохраняет диагностическое событие.
